Campus Event Management Platform

A Modern Solution for Student Engagement

Connecting students with campus events

Introduction / Problem Statement

Challenge: Campus students struggle to discover and engage with events

Current Issues:

  • Limited event discovery platforms
  • No centralized event management system
  • Poor student engagement with campus activities
  • Difficult event filtering and search

Solution: Create an Instagram-like platform for seamless event discovery and engagement

Objectives & Goals

  • Enable students to discover campus events effortlessly
  • Provide admins with complete event management control
  • Foster community engagement through interactions (likes, comments, reactions)
  • Implement secure authentication with email verification
  • Create an intuitive, mobile-friendly user interface
  • Track event engagement metrics and analytics

Features: User Experience

Authentication

Email verification with 6-digit codes, secure password protection

Event Browsing

Instagram-like feed with event photos, descriptions, and details

Interactions

Like, dislike, comment, and emoji reactions on events

Smart Filtering

Filter by category, sort by latest/oldest/popular

Search

Real-time search across event titles and descriptions

User Profile

View stats, liked events, and interaction history

Features: Admin Dashboard

CRUD Operations: Create, read, update, delete events

File Upload: Upload event images directly from device or use URLs

Analytics: View detailed engagement metrics for each event

User Insights: See who liked/disliked, all comments, reactions

Event Management: Organize by category, date, and location

Password Protection: Secure admin access with dedicated login

Architecture / Technology Stack

Frontend

React 19, Next.js 16, TypeScript

UI Framework

Tailwind CSS, shadcn/ui components

Backend

Next.js Server Actions, Route Handlers

Database

MongoDB with secure connection pooling

Authentication

HTTP-only cookies, bcryptjs hashing

Email Service

Resend for verification emails

Database Schema

Users Collection:

email, username, password (hashed), isVerified, likedPosts, dislikedPosts, createdAt

Events Collection:

title, description, imageUrl, category, location, date, likes[], dislikes[], comments[], reactions[], createdAt

Comments Schema:

_id, userId, username, text, createdAt

Reactions Schema:

userId, emoji

Challenges & Learnings

✓ Challenge 1: Implementing real-time email verification

Learned: Integrating third-party email services (Resend) with Next.js

✓ Challenge 2: Secure authentication without frameworks

Learned: Cookie-based sessions, password hashing, security best practices

✓ Challenge 3: Image file uploads and storage

Learned: Base64 encoding, file handling in Next.js, UX for image selection

✓ Challenge 4: Complex sorting and filtering

Learned: MongoDB aggregation, client-side filtering optimization
